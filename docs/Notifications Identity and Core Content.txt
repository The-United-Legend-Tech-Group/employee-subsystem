Identity and Core Content:

Recipient ID (recipientId): This is a required field that links the notification to the specific Employee (User) who needs to see it. It's an array allowing for multiple recepients to receive the notification.

Title (title): This is the required main headline or subject of the notification.

Message (message): This is the required main body content.

Classification and Delivery:

Type (type): This is required and defines the urgency or status of the alert. It must be one of the following: Alert, Info, Success, or Warning.

Delivery Type (deliveryType): This required field specifies the intended audience scope. The options are:

UNICAST: Intended for a single recipient.

MULTICAST: Intended for a specific group of recipients.

BROADCAST: Intended for all users.

Status and Related Data:

Is Read (isRead): A boolean (True/False) field that tracks whether the recipient has viewed the notification. It defaults to false (Unread).

Read At (readAt): An optional timestamp that records exactly when the notification was marked as read.

Related Entity ID (relatedEntityId): This optional field stores the ID of the specific item the notification refers to (e.g., the ID of a performance appraisal, a leave request, etc.).

Related Module (relatedModule): This optional field specifies the area of the application the notification came from (e.g., 'Performance', 'Leaves').

Technical Details:

Timestamps: Only the createdAt timestamp is automatically tracked (when the record was created).
